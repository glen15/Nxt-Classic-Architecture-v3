# 2. RandomTextApp - 업데이트 미션

### 미션: 앱 타이틀 변경

- 현재: `확신없는 랜덤 명언`
- 변경: `확신 넘치는 랜덤명언 by 홍길동`

> `홍길동` 부분에 자신의 이름을 넣으세요.

**힌트:** `client/src/App.tsx`

## 서버 업데이트

### 미션 1: 포트를 80으로 변경

- 서버가 80번 포트에서 실행되도록 변경하세요.

**힌트:** `server/.env`

### 미션 2: 명언 접미사 변경

- 현재: 명언 저장 시 `...아마도...` 가 자동으로 붙음
- 변경: `...!!!...` 가 붙도록 수정

**힌트:** `server/src/server.ts`

### 미션 3: 확신 있게 / 확신 없게 저장 버튼 분리

- 현재: 명언 저장 버튼이 1개
- 변경: `확신 있게 저장` 버튼과 `확신 없게 저장` 버튼 2개로 분리
- 확신 있게 저장하면 `...!!!...` 가 붙고, 확신 없게 저장하면 `...아마도...` 가 붙도록 구현하세요.

**구현 가이드:**

1. 클라이언트와 서버 사이에 "확신 여부"를 구분하는 약속을 정한다 (예: POST body에 어떤 필드를 추가할 것인지)
2. 서버 API가 그 값에 따라 접미사를 다르게 붙이도록 수정한다 (`server/src/server.ts`)
3. 클라이언트에 버튼 2개를 만들고, 각 버튼이 약속한 값을 서버에 전달한다 (`client/src/App.tsx`)
4. 두 버튼 각각 눌러서 결과를 확인한다

> AI 에이전트에게 작업을 맡길 때: "현재 POST /api/text로 명언을 저장하고 있다. body에 confident 필드를 추가해서, true이면 ...!!!... false이면 ...아마도...를 붙여라" 처럼 **API 약속을 먼저 정해서 전달**하면 클라이언트와 서버를 동시에 수정할 수 있습니다.

## 주의

> 명시적인 요구사항에 따라 발생하는 추가 변경사항들을 잘 확인하세요.

> 접미사 변경은 **새로 저장되는 명언**에만 적용됩니다. 기존 데이터의 `...아마도...`는 변경되지 않습니다. 기존 데이터를 수정하는 것은 완전히 다른 작업입니다.
